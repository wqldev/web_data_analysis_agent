# Agent 协作闭环

## ⚠️ 强制规则（主 Agent 必读）

**命中下方「适用场景」或同类表述 → 必须走本 Skill，不得绕过。**

| 禁止 | 必须 |
|------|------|
| 主 Agent 直接 `execute_query` 并输出报告 | 先 `create_task`，再依次委派三个 subagent |
| 跳过 task-board 状态跟踪 | 每阶段用 MCP 记录 plan / 结果 / 验收 |
| 自行判断「简单题」不走流程 | 由 verifier-agent 验收后再向用户交付 |

向用户交付的最终分析结论，应来自 **builder-agent 产出 + verifier-agent 验收通过**，而非主 Agent 临时查库。

## 统一报告结构

所有分析类交付物须符合 `.cursor/skills/agent-loop/report-template.md` 六节结构：

**1 分析概要 → 2 核心结论 → 3 关键指标 → 4 详细分析 → 5 业务建议 → 6 数据说明与局限**

主 Agent 在步骤 6「交付」时：**原样转述六节**，不重组、不删减章节。

## 闭环流程

```
用户需求（分析类）
  → 主 Agent：create_task（创建看板，记录 board_id）
  → Task(planner-agent)：拆解任务 + assign_task 写入看板
  → Task(builder-agent)：探库、SQL、生成分析报告
  → Task(verifier-agent)：独立验收
       ├─ passed → 返回主 Agent → 向用户交付六节报告
       └─ failed → request_rework → Task(builder-agent) → 重新验收（verifier 内循环，最多 2 轮）
```

## MCP 分工

| MCP | 谁用 | 用途 |
|-----|------|------|
| **task-board** | 主 Agent + 三个 subagent | 任务状态、plan、结果、验收、回修 |
| **mysql-local** | **仅 builder-agent** | list_tables / execute_query 取数 |

主 Agent **不得**在闭环完成前直接调用 mysql-local。

## task-board 工具

| 工具 | 用途 | 典型调用方 |
|------|------|-----------|
| `create_task` | 创建看板/任务 | 主 Agent / planner-agent |
| `assign_task` | 分配给 subagent | planner-agent |
| `list_tasks` | 查询任务状态（含 rework 信息） | 全部 |
| `append_result` | 记录执行/验收结果 | builder / verifier |
| `complete_task` | 标记任务完成 | builder / verifier |
| `request_rework` | verifier 打回 builder 并记录 fix_items | **verifier-agent** |

## 主 Agent 执行步骤

1. **识别**：命中适用场景 → 向用户简短说明「将按 planner → builder → verifier 闭环执行」
2. **启动**：`create_task(requirement=用户原话)`，记录 `board_id`
3. **规划**：`Task(planner-agent)`，传入用户需求 + `board_id`
4. **实现**：`Task(builder-agent)`，传入 `board_id` + planner JSON
5. **验收**：`Task(verifier-agent)`，传入 `board_id`（verifier 负责内部回修循环）
6. **交付**：verifier 返回 `passed: true` 后，按 report-template.md 六节原样转述 builder 报告；若 `passed: false` 且回修轮次用尽，向用户说明未通过项与 fix_items

## 失败处理

- planner 产出不清晰：补充 acceptance_criteria 后重跑 planner
- builder 未完成所有任务：继续 builder
- verifier 验收失败：**verifier 直接 Task(builder-agent) 回修**（`request_rework` 控制最多 2 轮），主 Agent 不介入回修循环
- 回修轮次用尽仍失败：主 Agent 向用户说明 fix_items，不强行交付
